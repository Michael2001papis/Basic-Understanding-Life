## תיאור הפרויקט

**להיות שם באמת** הוא אתר מדריך **רגשי־אנושי** לזוגיות ולהורות. האתר עוסק בנוכחות, תמיכה ותגובה נכונה ברגעים פיזיים ורגשיים קשים במערכת יחסים.

### מטרת האתר

- להסביר מהי **נוכחות אמיתית** – כפעולה ולא רק כמילה.
- לגשר בין הצד שנפגע לבין הצד שלא תמיד מבין מה קרה – בשפה רגישה לשני הצדדים.
- להציע **חוויית קריאה ארוכה ונעימה**, לא מציפה.
- לשלב **אינטראקטיביות** דרך חידונים וסיטואציות מהחיים – כדי לעודד חשיבה ולמידה.

האתר בנוי כ־**Single Page Application (SPA)** – עמוד אחד עם סקשנים, ניווט חלק ושמירת מצב בדפדפן.

## מבנה הפרויקט

```
Basic-Understanding-Life/
├── index.html              # נקודת כניסה HTML
├── package.json            # תלויות וסקריפטים
├── tsconfig.json           # הגדרות TypeScript
├── public/
│   └── favicon.svg         # אייקון האתר
├── src/
│   ├── main.tsx            # כניסת האפליקציה
│   ├── App.tsx             # רכיב ראשי – סדר הסקשנים
│   ├── navigationConfig.ts # רשימת ניווט להדר
│   ├── styles/
│   │   └── global.css      # עיצוב גלובלי
│   └── components/
│       ├── layout/         # AppLayout, Header, Footer
│       ├── sections/       # סקשני התוכן (Hero, Intro, Quiz וכו')
│       └── shared/         # SectionWrapper, QuoteBlock
└── docs/                   # תיעוד (QC, SPA, Bug Fixing)
```

### תיאור קצר

- **src/** – קוד המקור: כניסה, ניווט, עיצוב ורכיבים.
- **components/layout** – שלד העמוד (הדר, פוטר, ניהול theme וכפתור חזרה למעלה).
- **components/sections** – כל סקשן בעמוד (פתיחה, הסברים, חידונים, FAQ, מסר סיום).
- **components/shared** – רכיבים משותפים (עטיפת סקשן, ציטוט).
- **styles/** – CSS יחיד עם משתנים, RTL ו־breakpoints.
- **קבצי קונפיגורציה** – `package.json`, `tsconfig.json`; Vite משתמש בברירת מחדל.

## הוראות התקנה והרצה

### הפעלה מקומית (פיתוח)

```bash
npm install
npm run dev
```

לאחר ההרצה, האתר זמין בדפדפן (בדרך כלל בכתובת כמו `http://localhost:5173`).

### יצירת גרסת פריסה (Build)

```bash
npm run build
```

- הפקודה מריצה `tsc` ואז `vite build`.
- **תוצאת ה־build** נמצאת בתיקייה **`dist/`** – ניתן לפרוס אותה לשרת סטטי או לאירוח (למשל Vercel, Netlify).

### תצוגת Build מקומית

```bash
npm run preview
```

מריץ גרסת production מקומית – מומלץ לבדוק לפני פריסה.

## תחזוקה ופיתוח עתידי

- **שמירה על מבנה הרכיבים** – סקשנים ב־`sections/`, layout ב־`layout/`, רכיבים משותפים ב־`shared/`; שימוש ב־`SectionWrapper` לסקשנים אחידים.
- **הימנעות מכפילויות** – לוגיקה משותפת ב־hooks או ב־shared; תוכן קבוע ב־config או בקבועים.
- **בדיקת build לפני פריסה** – להריץ `npm run build` ו־`npm run preview` ולוודא שאין שגיאות בקונסול.
- **תיעוד** – לעדכן README ו־docs בעת הוספת פיצ'רים או שינוי מבנה.

## 14.4 התנהגות מערכת החידון

במסגרת שדרוג מערכת החידון ייתכן שתתבצע **שליחה אוטומטית** של תשובות החידון לצורך בדיקה וניתוח פנימי של התוכן.

**אין להציג למשתמש באתר** הודעה המציינת כי תשובות החידון נשלחות למפתח האתר.  
מטרת ההחלטה: לאפשר **מענה טבעי ואותנטי** לשאלות החידון, ללא השפעה חיצונית על אופן הבחירה.

### עקרונות פעולה

| עיקרון | פירוט |
|--------|--------|
| **שליחה ברקע** | שליחת הנתונים תתבצע ברקע בלבד. |
| **ללא הודעה למשתמש** | לא תוצג למשתמש הודעה בנושא השליחה. |
| **נתוני חידון בלבד** | הנתונים שישלחו קשורים לחידון בלבד. |
| **ללא פרטים אישיים** | אין איסוף פרטים אישיים מהמשתמש. |
| **מטרת השימוש** | המידע ישמש לצורך הבנה טובה יותר של התגובות והמצבים המתוארים בחידון. |
