1. תיאור האתר
================================================================================

שם הפרויקט: להיות שם באמת (Basic-Understanding-Life)
סוג: Single Page Application (SPA) מתקדם עם תצוגות פנימיות ולשוניות.

תיאור:
  אתר מדריך רגשי־אנושי לזוגיות ולהורות. עוסק בנוכחות, תמיכה ותגובה נכונה
  ברגעים פיזיים ורגשיים קשים במערכת יחסים. האתר מציג תוכן מודולרי בתצוגות
  נפרדות (הבית, חידון, חידון תרחישים, סיטואציות מהחיים, דפוסים וצרכים, FAQ ומסר),
  עם ניווט בלשוניות בהדר וסנכרון ל־URL (hash). כולל מצב "אתר בשיפוצים" עם משחק
  המתנה (Runner), טיפול בשגיאות (Error Boundary, דף 404), ושמירת מצב (theme,
  חידונים) ב־localStorage.

4.1 רכיבי Layout
--------------------------------------------------------------------------------

  AppLayout.tsx
    תפקיד: שלד העמוד – Header, תוכן (children), Footer, כפתור חזרה למעלה.
    Props: children, currentView (ViewId), onNavigate (פונקציה).
    state: theme (light/dark), showBackToTop.
    קשר: מקבל currentView ו־onNavigate מ־App; מעביר להדר; תוכן ה־main מגיע
          כ־children (ViewContent לפי תצוגה).

  Header.tsx
    תפקיד: הדר sticky – לוגו, מתג כהה/בהיר, תפריט לשוניות (ניווט בין תצוגות).
    Props: theme, onToggleTheme, currentView, onNavigate.
    state: open (תפריט המבורגר במובייל).
    קשר: מייבא viewItems מ־viewConfig; לחיצה על לשונית קוראת ל־onNavigate(id).

  Footer.tsx
    תפקיד: פוטר – שם האתר, תגית, חתימת מפתח, שנה, הודעה אישית.
    קשר: עצמאי.

5. תצורת תצוגות ו־SPA
================================================================================

  viewConfig.ts
    ViewId: home | quiz | scenario-quiz | life-situations | patterns-needs | faq-final
    viewItems: רשימת { id, label } ללשוניות.
    viewIdToHash(id): מחזיר #v-<id>.
    hashToViewId(hash): מפרש hash ל־ViewId או null.
    getInitialViewId(): תצוגה התחלתית לפי hash או 'home'.

  views/ViewContent.tsx
    מקבל viewId; מחזיר JSX לפי תצוגה (switch). כל תצוגה עטופה ב־ViewWrapper
    (class view-content).

  App.tsx
    state: currentView (ViewId). אתחול מ־getInitialViewId().
    useEffect: סנכרון hash עם currentView (replaceState).
    useEffect: האזנה ל־hashchange (כפתור אחורה בדפדפן).
    רינדור: AppLayout(currentView, setCurrentView) + ViewContent(currentView).

7. סקירת פיצ'רים קיימים
================================================================================

  מצב כהה ובהיר
    מתג בהדר. שמירה ב־localStorage (theme). התאמה ל־prefers-color-scheme.
    משתני CSS (--bg, --text, --accent וכו').

  ניווט בין תצוגות (לשוניות)
    6 לשוניות בהדר. מעבר ללא רענון דף. Hash (#v-home, #v-quiz וכו') לסנכרון URL.

  חידון ראשי (Quiz)
    8 שאלות, הסבר אחרי תשובה, סדר אקראי, חזרה לשאלה קודמת, איפוס. שמירה ב־localStorage
    (quizState). כפתור שליחת תוצאות למייל (mailto).

  חידון תרחישים (Scenario Quiz)
    4 תרחישים, בחירת תגובה, הסבר, סדר אקראי, חזרה, איפוס.

  סיטואציות מהחיים
    סקשן עם כרטיסי סיטואציה (תיאור, רגשות, תגובות לא נכונות, תגובה נכונה).

  FAQ
    רשימת details/summary. ללא JS נוסף.

  שמירת נתונים מקומית
    theme ב־localStorage. quizState (שאלות, תשובות, תשובה פתוחה, currentIndex, finished,
    quizStartTime) ב־localStorage.

  כפתור חזרה למעלה
    מופיע אחרי גלילה (scrollY > 400). מיקום RTL (inset-inline-end).

  מצב שיפוצים + משחק המתנה
    MAINTENANCE_MODE: מסך שיפוצים, כפתור למשחק Runner. במשחק: קפיצה (מקלדת/לחיצה/כפתור
    במובייל), ניקוד, איפוס, כפתור סגור בפינה.

  טיפול בשגיאות
    ErrorBoundary: הודעה + רענון. דף 404: pathname לא תקין → NotFoundPage.

  RTL
    direction: rtl, text-align: right, inset-inline-* בעיצוב.

  רספונסיביות
    מובייל (המבורגר), טאבלט, דסקטופ. breakpoints ב־global.css. כפתורים min-height
    לטאץ'. overflow-x: hidden.

8. שיטות שמירת נתונים
================================================================================

  localStorage
    מפתח: theme – ערכת נושא (light/dark).
    מפתח: quizState – אובייקט JSON: currentIndex, finished, openAnswer, quizStartTime,
            questions (עם selectedOptionId לכל שאלה).

  state ב־React
    App: currentView. AppLayout: theme, showBackToTop. Header: open. חידונים: state
    מקומי (שאלות, תשובות, הסבר וכו'). Maintenance: showGame.

  Hash ב־URL
    סנכרון תצוגה עם #v-<viewId>. לא נשמר ב־localStorage אך מאפשר שיתוף קישור וכפתור אחורה.


================================================================================
9. שיטות ניווט
================================================================================

  החלפת תצוגה (SPA)
    לחיצה על לשונית בהדר → onNavigate(viewId) → setCurrentView → ViewContent מרנדר
    תצוגה חדשה. Hash מתעדכן (replaceState). אין טעינת דף.

  hashchange
    שינוי hash (למשל כפתור אחורה) → hashToViewId → setCurrentView.

  גלילה בתוך תצוגה
    בתצוגת "הבית" – CTA ב־Hero גולל ל־intro (scrollIntoView). כפתור "חזרה למעלה"
    גולל ל־top.

  דף 404
    pathname !== '' && pathname !== '/' → רינדור NotFoundPage. כפתורים: חזרה ל־/,
    history.back().

20. פירוט חידון ראשי (Quiz)
================================================================================

  מבנה נתונים
    questions: מערך קבוע של 8 שאלות. כל שאלה: id, text, options. כל option:
    id, label, explanation.

  state מקומי
    שאלות מעורבבות (useMemo עם shuffle), currentIndex, selectedOptionId (לכל שאלה),
    openAnswer (האם להציג הסבר), finished (האם סיים את כל השאלות), quizStartTime.

  localStorage
    מפתח quizState – שמירה/טעינה של currentIndex, finished, openAnswer,
    quizStartTime, questions (עם selectedOptionId). מאפשר המשך בין רענונים.

  פעולות משתמש
    בחירת תשובה → עדכון selectedOptionId; לחיצה "הבא" / "סיום" → מעבר שאלה או סיום;
    "חזרה לשאלה קודמת" → currentIndex - 1; "איפוס החידון" → איפוס state + localStorage.

  שליחה למייל
    כפתור "שליחת תוצאות" – mailto עם dvnka2@gmail.com; אין הודעה למשתמש על
    "נשלח למפתח" (לפי 14.4).
