# 12. הבטחת מבנה SPA מלא (Single Page Application)

האתר נבנה כ־**Single Page Application** באמצעות React + TypeScript + Vite. מסמך זה מתאר את העקרונות, היישום והבדיקות לווידוא שהאתר מתנהג במלואו כ־SPA ומציג חוויית שימוש מודרנית, חלקה ודינמית.

---

## 12.1 עקרונות SPA שנשמרים

| עיקרון | יישום באתר |
|--------|------------|
| **טעינה ראשונית אחת** | `index.html` נטען פעם אחת; כל התוכן נטען דרך React (`main.tsx` → `App.tsx` → רכיבים). אין דפים נפרדים. |
| **ניווט ללא רענון דף** | מעבר בין סקשנים מתבצע ב־JavaScript בלבד (כפתורים בתפריט → `scrollIntoView({ behavior: 'smooth' })`). אין קישורים שגורמים ל־Full Page Reload. |
| **ניהול מצב בצד הלקוח** | theme ו־quiz state ב־React state; שמירה ב־localStorage (theme, quizState). אין תלות בשרת לניווט או למצב. |
| **עדכון DOM דינמי** | React מעדכן רק את החלקים הנדרשים (state → re-render). אין יצירה מחדש של עמודים בעת גלילה או ניווט. |

---

## 12.2 מניעת תחושה של אתר HTML סטטי

כדי שהאתר ירגיש כאפליקציה אינטראקטיבית:

- **טעינה חלקה** – הסקשנים נטענים כחלק מעץ React אחד; גלילה רציפה.
- **גלילה חלקה** – `scrollIntoView({ behavior: 'smooth' })` בכפתורי הניווט ו־CTA.
- **אינטראקציות דינמיות** – החלפת theme, חידונים, FAQ מתקפל, כפתור חזרה למעלה, **הדגשת הסקשן הפעיל בתפריט** (Intersection Observer).
- **שמירת מצב** – theme ו־משחקון (כולל תשובות ותשובה פתוחה) נשמרים ב־localStorage; אחרי רענון המצב משוחזר.

---

## 12.3 בדיקות לווידוא SPA

| בדיקה | אופן האימות |
|-------|--------------|
| מעבר בין חלקי האתר אינו גורם לרענון | לחיצה על פריטי תפריט – הדף לא נטען מחדש; רק גלילה. |
| אין טעינת עמודים מחדש בניווט | אין שימוש ב־`<a href="...">` שמפנה לאותו דומיין עם reload; הניווט מבוסס על `button` + `scrollIntoView`. |
| מצב האתר נשמר בשימוש רגיל | theme ו־quiz state נשמרים ב־localStorage; גלילה לא מאפסת אותם. |
| קונסול הדפדפן נקי משגיאות | להריץ `npm run build`, לפתוח את האתר, לבדוק Console – אין שגיאות JavaScript. |

---

## 12.4 מטרה

המטרה היא שהאתר ירגיש למשתמש כ־**אפליקציה אינטראקטיבית מודרנית**, ולא כעמוד HTML סטטי.

גישה זו משפרת:

- **חוויית משתמש** – תגובה מיידית, גלילה חלקה, סקשן פעיל מודגש.
- **מהירות תגובה** – אין המתנה לטעינת דף חדש.
- **תחושת מודרניות** – ממשק אחיד ויציב.
- **יציבות הממשק** – state צפוי ולא אובד בגלילה או בפעולות.

---

## טכניקות יישום

- **זיהוי סקשן פעיל:** `IntersectionObserver` ב־`AppLayout` עוקב אחרי כל הסקשנים (לפי `navItems`); הסקשן הראשון שנכנס ל־viewport (עם `rootMargin` מתאים) מגדיר את `activeSectionId`, שמועבר ל־`Header` ומדגיש את הפריט המתאים בתפריט (class `nav-item-active`).
