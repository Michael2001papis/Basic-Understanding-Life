# 13. תיקון באגים וייצוב המערכת (Bug Fixing & Stability)

במהלך המשך העבודה על האתר יבוצע תהליך מסודר של איתור ותיקון באגים, במטרה לשמור על אתר יציב, מהיר ואמין עבור המשתמשים.

תהליך זה יכלול בדיקה מקיפה של כל רכיבי האתר: קוד, מבנה, ניווט, פיצ'רים אינטראקטיביים וחוויית משתמש.

---

## 13.1 סריקת באגים מלאה

תבוצע סריקה כללית של האתר במטרה לאתר:

| תחום | מה בודקים |
|------|------------|
| **שגיאות JavaScript** | קונסול הדפדפן, שגיאות runtime |
| **בעיות React rendering** | אזהרות ב־StrictMode, רכיבים שלא מתעדכנים |
| **תקלות ניווט** | מעבר בין סקשנים, הדגשת סקשן פעיל, תפריט המבורגר |
| **בעיות רספונסיביות** | שבירת layout, גלילה אופקית, כפתורים קטנים מדי |
| **תקלות בחידונים** | בחירת תשובות, הסברים, שמירה ב־localStorage, שליחת מייל |
| **טעינה וגלילה** | זמן טעינה, גלילה חלקה, כפתור חזרה למעלה |

### סיווג חומרה

- **קריטית** – פיצ'ר לא עובד, אתר לא נטען, שגיאה חוסמת.
- **בינונית** – פיצ'ר חלקי, עיצוב שבור במסך מסוים, חוויית משתמש פגומה.
- **קלה** – שיפור עיצובי, טקסט, נגישות או עקביות קוד.

## 13.2 תיקון בעיות קוד

במסגרת תיקון הבאגים ייבדקו:

- **כפילויות קוד** – לוגיקה חוזרת שניתן לאחד.
- **משתנים או פונקציות שלא בשימוש** – להסיר או לייצא אם נדרש.
- **בעיות לוגיקה** – תנאים, state, תלות בין רכיבים.
- **בעיות TypeScript** – טיפוסים חסרים, strict mode.
- **עקביות עם מבנה הפרויקט** – תיקיות sections/layout/shared, שימוש ב־SectionWrapper.

**מטרה:** קוד נקי, ברור וקל לתחזוקה.

## 13.7 בדיקות Build ופריסה

לפני כל פריסה של האתר:

1. **Build מלא:**
   ```bash
   npm run build
   ```
   חובה שיסתיים בהצלחה (exit code 0).

2. **תצוגת פריסה מקומית:**
   ```bash
   npm run preview
   ```
   לבדוק את האתר בדפדפן – ניווט, theme, חידונים.

3. **קונסול:** לוודא שאין שגיאות JavaScript בקונסול הדפדפן.

## 13.9 שיפור מערכת החידון (Quiz Improvements)

יושמו השיפורים הבאים:

### 13.9.1 גיוון סדר התשובות (Randomized Order)

- סדר האפשרויות בכל שאלה מתערבב (Fisher-Yates) בכל טעינה של החידון ובכל איפוס.
- מונע חזרתיות, מעודד בחירה מתוך חשיבה ולא לפי מיקום.

### 13.9.2 כפתור "חזרה לשאלה קודמת"

- מופיע כשהמשתמש לא בשאלה הראשונה (גם במסך השאלה וגם במסך ההסבר).
- מאפשר חזרה לשאלה הקודמת ולשינוי תשובה.

### 13.9.3 כפתור "איפוס החידון"

- מופיע במסך השאלות ובמסך הסיום.
- מוחק את נתוני החידון מ־localStorage, מאפס את המצב ומתחיל מהשאלה הראשונה עם סדר תשובות חדש.

### 13.9.4 יציבות

- ללא שינוי ב־state logic; מבנה החידון נשמר; תאימות למובייל ולמצב כהה/בהיר.

## 14.5 שיפור חידון התרחישים (Scenario Quiz)

יושמו השיפורים הבאים:

### 14.5.1 סדר דינמי של אפשרויות התשובה

- אפשרויות התשובה בכל תרחיש מתערבבות (Fisher-Yates) בכל טעינה ובכל איפוס – מפחית חזרתיות ומעודד חשיבה על הסיטואציה.

### 14.5.2 כפתור "חזרה אחורה"

- מאפשר חזרה לתרחיש הקודם ולשינוי תשובה.

### 14.5.3 כפתור "איפוס החידון"

- מאפס את מצב החידון ומתחיל מהתרחיש הראשון עם סדר תשובות חדש. (חידון התרחישים לא שומר state ב־localStorage – במידה שיוסף בעתיד, האיפוס ימחק גם אותו.)

### 14.5.4 יציבות

- יציבות React state; התאמה למובייל ולטאבלט; תמיכה במצב כהה ובהיר; שמירה על מבנה הקוד.

## 15. מצב "אתר בשיפוצים" עם משחק המתנה

### הפעלה

- בקובץ `src/maintenanceConfig.ts`: `MAINTENANCE_MODE = true` → מוצג מסך שיפוצים במקום האתר. `false` → האתר פועל כרגיל.

### מסך שיפוצים

- הודעה: האתר בשיפוצים, מתעדכן וישוב בהקדם.
- כפתור: "משחק להעברת הזמן" – פותח משחק Runner.

### משחק ההמתנה

- דמות (ריבוע) שקופצת מעל מכשולים; התקדמות אוטומטית; ניקוד בזמן אמת.
- שליטה: ריווח או לחיצה לקפיצה.
- כפתורים: איפוס (מאפס ומתחיל מחדש), חזרה למסך השיפוצים.

### 16. התאמת שליטת המשחק במובייל ובטאבלט

- **כפתור קפיצה (↑):** מוצג רק במסכים עד 1024px (מובייל וטאבלט). במחשב – שליטה במקלדת (רווח/חץ למעלה) או לחיצה על הקנבס.
- **מאפייני הכפתור:** גודל 3.5rem (נוח לאצבע), מיקום בפינה השמאלית התחתונה של אזור המשחק, `touch-action: manipulation` למניעת עיכוב ולחיצות כפולות לא רצויות.
- **חוויית משחק:** תגובה מיידית ללחיצה; אותה לוגיקת קפיצה (רק כשהדמות על הקרקע).

## 15.6 טיפול בשגיאות והודעות מערכת ברורות

כחלק משיפור יציבות האתר וחוויית המשתמש, נוספה מערכת לטיפול בשגיאות (Error Handling) שמספקת למשתמש הודעות ברורות ומובנות כאשר מתרחשת תקלה או כאשר המשתמש מגיע לעמוד שאינו קיים. מטרת המערכת: למנוע בלבול ולהציע פתרונות ברורים במצבי שגיאה.

### 15.6.1 דף 404 – עמוד לא נמצא

כאשר משתמש נכנס לכתובת שאינה קיימת באתר (קישור שבור, כתובת שגויה), מוצג דף שגיאה ייעודי:

- הודעה כי הדף המבוקש אינו קיים.
- הסבר קצר על סיבה אפשרית (קישור שבור, כתובת שגויה, עמוד הוסר).
- כפתור "חזרה לעמוד הראשי".
- כפתור "חזרה לעמוד הקודם".

(ב־SPA: בדיקת `pathname` ב־main – אם שונה מ־`/` מוצג `NotFoundPage`.)

### 15.6.2 הודעות שגיאה למשתמש

במצבי תקלה (טעינת תוכן, חידון, שמירת נתונים, ניווט) מוצגות הודעות קצרות, ברורות וידידותיות – לא הודעות טכניות.

### 15.6.3 מניעת הודעות טכניות

לא מוצגות למשתמש הודעות כמו `TypeError` או `Unhandled exception`. במקום זאת: "אירעה תקלה זמנית", "לא ניתן להציג את התוכן כרגע. נסה לרענן את הדף" וכפתור רענון (Error Boundary).

### 15.6.4 יציבות

- `ErrorBoundary` תופס שגיאות JavaScript ב־React ומונע קריסת כל העמוד.
- המשתמש רואה מסך החלפה ידידותי ויכול לרענן או לחזור.
- דף 404 מחזיר את המשתמש למסלול שימוש תקין (עמוד ראשי או קודם).

### 15.6.5 מטרת המנגנון

- לשפר יציבות האתר.
- להציג מידע ברור במצבי תקלה.
- למנוע בלבול ולשמור על חוויית משתמש מקצועית.
